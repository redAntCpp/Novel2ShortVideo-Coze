# 编译层介绍

Compiler，将结构化的分镜脚本，逐个编译，生成可以直接使用的即梦提示词。从“导演数据”编译成“Seedance导演语言”。这也是整个系统真正最有技术含量的地方之一。

提示词不是简单的：

```tex
JSON
↓
字符串拼接
↓
Prompt
```

真正的 Prompt Compiler 分层而是：

```tex
1. 全局风格注入
2. 场景空间注入
3. 角色引用注入
4. 时间分镜编译
5. 镜头语言编译
6. 表演语言编译
7. 声音语言编译
8. 约束条件编译
9. Seedance语法优化
```

Prompt 不是一段文字,而是多层信息融合，最后输出的应该是：

```tex
镜头1：
……

镜头2：
……

镜头3：
……
```

因为官方已经明确建议：

> 使用 镜头1、镜头2、镜头3 等标识，按事件发生顺序组织内容。

这就意味着，Seedance 对“时间化分镜结构”有特殊优化

因此，我们不应该把整个 JSON 一次性喂 LLM。而应该逐 shot 编译。因为shot 才是最小导演单元。

因此：整个流程应该是：

```tex
for shot in shots:

    1. 注入 global_style
    2. 注入 scene
    3. 注入角色定义
    4. 编译 camera
    5. 编译 action timeline
    6. 编译 performance
    7. 编译 audio
    8. 编译 constraints

    ↓

    输出：
    当前shot完整导演prompt
```

## 编译示例

```json
{
  "shot_id": 1,

  "subjects": ["linxia"],

  "camera": {
    "shot_type": "medium shot",
    "movement": "slow push in",
    "stability": "handheld subtle shake",
    "lens": "50mm"
  },

  "action": {
    "timeline": [
      {
        "start": 0,
        "end": 1.5,
        "description": "林夏低头盯着可乐罐发呆"
      },
      {
        "start": 1.5,
        "end": 3,
        "description": "她缓慢抬头看向便利店玻璃门"
      }
    ]
  }
}
```

### 第一步：空间层编译

先把：

- global_style
- scene
- character

融合。如果有参考图，直接使用地址即可。

```tex
深夜便利店，
冷白色日光灯，
窗外暴雨，
冷蓝灰白色调，
日式悬疑电影风格。

林夏身穿白衬衫，
黑长发，
佩戴细框眼镜，
神情疲惫。
```

这里还是静态空间层。

### 第二步：时间层编译

加入：timeline

得到：

```tex
镜头1：

0s-1.5s：林夏站在便利店货架前，
低头盯着手中的可乐罐发呆。

1.5-3s：随后她缓慢抬头，
看向便利店玻璃门。
```

这里得到：时间推进

### 第三步：镜头语言编译

翻译成摄影语言。

例如：

```json
{
  "movement": "slow push in"
}
```

翻译成：镜头缓慢推近

再例如：

```json
{
  "stability": "handheld subtle shake"
}
```

翻译成：带有轻微真实手持晃动感

### 第四步：情绪与表演编译

performance，例如

```json
{
  "facial_expression": "疲惫压抑"
}
```

不要直译，而是要：她神情压抑疲惫，眼神空洞。

### 第五步：音频编译

例如：

```
{
  "ambient": [
    "暴雨声",
    "冰柜电流声"
  ]
}
```

翻译成：背景持续传来暴雨声与冰柜低沉电流嗡鸣。

### 第六步：最终翻译

示例：

```tex
镜头1：

深夜便利店内，
冷白色日光灯映照着潮湿地面，
窗外暴雨不断冲刷玻璃。

林夏身穿白衬衫，
黑色长发，
佩戴细框眼镜，
独自站在货架前。

0s-1.5s:她低头盯着手中的可乐罐发呆，
1.5-3s:随后缓慢抬头，看向便利店玻璃门。

中景镜头，
50mm电影镜头语言，
镜头缓慢推近，
带有轻微真实手持晃动感。

整体呈现冷蓝灰白色调，
日式悬疑电影氛围，
35mm胶片颗粒质感。

背景持续传来暴雨声与冰柜低沉电流嗡鸣。
```

## 核心规则

```tex
你是一名 AI 视频导演 Prompt 编译器。

你的职责不是“创作剧情”，
而是将结构化 Storyboard JSON
编译为适用于 Seedance 2.0 的电影级导演提示词。

你必须严格遵循以下原则：

# 核心原则

1. 不新增剧情
不得虚构 JSON 中不存在的事件。

2. 不改变人物
必须严格保持角色定义一致。

3. 不改变场景
所有镜头必须继承 scene 与 global_style。

4. 强化时间顺序
必须明确描述动作的先后顺序。

5. 强化镜头语言
必须将 camera 字段翻译为自然电影摄影语言。

6. 强化视觉可见性
优先描述“可被看见”的内容，
而非抽象情绪词。

错误示例：
“她很悲伤”

正确示例：
“她低头沉默，眼眶泛红”

7. 使用分镜格式
输出必须按：

镜头1：
镜头2：
镜头3：

组织。

8. 保持电影语言
输出风格应像：
导演分镜脚本
而不是小说。

9. 环境持续性
必须持续体现：
天气
灯光
环境细节
空间氛围。

10. 不使用解释性语言
不要分析。
不要说明。
直接输出导演提示词。

# Camera 编译规则

slow push in
→ 镜头缓慢推近

whip pan
→ 镜头快速甩动

tracking shot
→ 镜头持续跟拍

handheld
→ 带有真实手持晃动感

# 时间层规则

timeline 必须体现：
先发生什么
后发生什么。

不得压缩成一句静态描述。

# 输出目标

输出结果必须：

1. 可直接用于 Seedance 2.0
2. 具有明确镜头感
3. 具有连续时间感
4. 具有统一电影风格
5. 具有空间连续性
```
