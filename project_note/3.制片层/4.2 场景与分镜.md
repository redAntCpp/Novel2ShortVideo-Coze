# 场景与分镜

## 场景

场景是空间规则系统。主要包括

| 内容     | 作用         |
| -------- | ------------ |
| 地点     | 在哪         |
| 时间     | 白天还是深夜 |
| 天气     | 雨雪雾       |
| 环境细节 | 空间真实感   |
| 氛围     | 情绪基调     |
| 声音     | 环境音       |
| 空间布局 | 镜头一致性   |

示例：

```json
{
	"scene": {
		"scene_id": "id",
		"reference_image": "scenes/bianlidian.png",
		"location": "深夜便利店",
		"weather": "暴雨",
		"time": "凌晨",
		"environment_details": [
			"窗外雨水流淌",
			"便利店货架冷白灯光",
			"冰柜轻微嗡鸣",
			"地面潮湿反光"
		],
		"layout": {
			"door": "right",
			"cashier": "left",
			"window": "front"
		},
		"scene_assets": [
			"便利店货架",
			"冰柜",
			"自动门"
		]
	}
}
```

### location

简短的地点介绍

### weather

天气 = 动态氛围生成器

| 天气 | 增强什么   |
| ---- | ---------- |
| 暴雨 | 紧张、反光 |
| 大雪 | 孤独       |
| 大雾 | 神秘       |
| 晴天 | 明亮       |

AI 会自动生成：

- 雨声
- 水滴
- 潮湿反光
- 雨伞
- 雨水运动

这就是：空间层影响时间层

### time
时间决定光影逻辑。

| 时间 | 光线     |
| ---- | -------- |
| 清晨 | 柔光     |
| 正午 | 强硬阴影 |
| 黄昏 | 金色     |
| 深夜 | 高反差   |

比如

```tex
"time": "凌晨2点"
```

会自动影响：人少/店空/光线冷/氛围安静

### environment_details

AI 视频像假 CG，环境缺少微细节。

```json
[
  "窗外雨水流淌",
  "冰柜轻微嗡鸣",
  "地面潮湿反光"
]
```

### layout

环境布局

```jso
{
  "layout": {
    "door": "right",
    "cashier": "left",
    "window": "front"
  }
}
```

### scene_assets

场景资产，要符合当下时代背景

```json
{
  "scene_assets": [
    "便利店货架",
    "冰柜",
    "自动门"
  ]
}
```

## shots

shot 不是一个镜头。shot = 一个连续的时空事件。

不是画面，而是事件，真正的视频生成，本质是在生成“变化”。

| 类型     | 示例        |
| -------- | ----------- |
| 人物变化 | 转头、奔跑  |
| 镜头变化 | 推近、摇镜  |
| 光线变化 | 爆炸火光    |
| 情绪变化 | 平静 → 恐惧 |
| 场景变化 | 门被撞开    |
| 声音变化 | 风铃突然响  |

```tex
shot
├── subjects
├── camera
├── action timeline
├── performance
├── dialogue
├── audio
├── transition
├── emotion curve
├── timing
└── seedance compilation
```

示例如下：

```json
{
      "shot_id": 1,
      "duration": 3,

      "purpose": "建立孤独压抑氛围",

      "subjects": ["linxia"],

      "camera": {
        "shot_type": "medium shot",
        "movement": "slow push in",
        "stability": "handheld subtle shake",
        "lens": "50mm",
        "framing": "center composition"
      },

      "action": {
        "timeline": [
          {
            "start": 0,
            "end": 1.5,
            "description": "林夏低头盯着手里的可乐罐发呆"
          },
          {
            "start": 1.5,
            "end": 3,
            "description": "她缓慢抬头，看向便利店玻璃门"
          }
        ]
      },

      "performance": {
        "facial_expression": "疲惫压抑",
        "body_language": "肩膀轻微下沉"
      },

      "audio": {
        "bgm": {
          "style": "低沉钢琴氛围",
          "intensity": "low"
        },
        "ambient": [
          "暴雨声",
          "冰柜电流声"
        ],
        "sfx": []
      },
    }
```

### duration

不只是秒数，是节奏控制器。

| 时长  | 感觉   |
| ----- | ------ |
| 0.5秒 | 紧张   |
| 2秒   | 正常   |
| 8秒   | 文艺   |
| 15秒  | 长镜头 |

### purpose

镜头说明：这个镜头为什么存在

purpose 可以帮助：

- Prompt Compiler
- 镜头 Agent
- 音乐 Agent

例如：

```json
"purpose": "建立角色孤独"
```

### subjects 

镜头参与者

```json
"subjects": ["linxia"]
```

### camera

真正决定电影感的，是 camera

```json
"camera": {
  "shot_type": "medium shot",
  "movement": "slow push in",
  "stability": "handheld subtle shake",
  "lens": "50mm"
}
```

- 分镜类型（shot_type）

  | **术语（提示词）**                 | **对应中文**  | **画面效果（以拍人为例）**         | **常见用途**                         |
  | ---------------------------------- | ------------- | ---------------------------------- | ------------------------------------ |
  | **`Extreme Long Shot (ELS)`**      | 大全景 / 远景 | 人物变成了小点，全是风景。         | 交代故事发生的大环境、大背景。       |
  | **`Long Shot (LS)` / `Full Shot`** | 全景          | 拍到人物的全头全脚，能看清全身。   | 表现人物在环境中的位置和肢体动作。   |
  | **`Medium Shot (MS)`**             | 中景          | 拍到**腰部**以上。                 | 对话戏的主力，最自然的叙事视角。     |
  | **`Medium Close-up (MCU)`**        | 中特写        | 拍到**胸部**或肩膀以上。           | 开始把焦点聚在面部，捕捉微妙的表情。 |
  | **`Close-up (CU)`**                | 特写          | 只拍**整张脸**，或者某个特写局部。 | 放大情绪，表现紧张、痛苦或震惊。     |
  | **`Extreme Close-up (ECU)`**       | 大特写        | 只拍一只眼睛、嘴唇或者手指。       | 极具视觉冲击力，强调某一个关键细节。 |

- 运镜（movement），可以配合缓慢、快速等。

  基础前后左右移动：

  - **`push in` / `zoom in`**：推镜头。画面慢慢向前靠近主体。（注：`push` 是机位动，`zoom` 是变焦，AI 里常混用，但 `push` 往往更真实）。
  - **`pull out` / `zoom out`**：拉镜头。画面慢慢向后退，露出更多环境。
  - **`pan left / pan right`**：摇镜头。机位不动，镜头像人转头一样往左或往右看。
  - **`tilt up / tilt down`**：俯仰镜头。机位不动，镜头垂直往上看或往下看。

  高级轨迹移动：

  - **`truck left / truck right`**：横移。摄像机平行于主体，跟着主体一起往左或往右平行移动。
  - **`crane shot` / `pedestal up`**：升降镜头。镜头像坐在吊车上一样垂直向上升起（通常用来做震撼的结尾或开场）。
  - **`orbit shot` / `360-degree pan`**：环绕运镜。镜头围着主角转圈圈，非常有戏剧张力

- stability（手持）： 决定真实感，比如

  | 稳定性   | 感觉 |
  | -------- | ---- |
  | tripod   | 冷静 |
  | handheld | 真实 |
  | shaky    | 恐慌 |

- lens（焦段）

  | 焦段 | 感觉     |
  | ---- | -------- |
  | 24mm | 空间夸张 |
  | 35mm | 电影感   |
  | 50mm | 人眼     |
  | 85mm | 人像压缩 |

### action.timeline 

进入时间层。

正常来说，```"action": "男人奔跑"```,而是应该：

```json
"timeline": [
  {
    "start": 0,
    "end": 1,
    "description": "男人缓慢起跑"
  },
  {
    "start": 1,
    "end": 2,
    "description": "撞翻水果摊"
  }
]
```

timeline 就是在定义：状态变化曲线

### performance

AI 更擅长“动作”，而不是“抽象情绪”。

```json
"performance": {
  "facial_expression": "眼眶泛红",
  "body_language": "手指微微颤抖"
}
```

而不是：

```json
"emotion": "悲伤"
```

### dialogue

Seedance 是原生音视频联合,所以

```json
{
  "type":"speaking",
  "speaker": "chenmo",
  "time": 3,
  "text": "他们找到我了"
}
```

type:

- speaking：正常说话
- singing：唱（包括戏曲、歌曲）
- chanting：念（戏曲念白、仪式语言）
- voice_over：内心独白
- narration: 旁白

speaker：

说话的主体，当type = narration时，可以为空。

### audio
在 Seedance 时代，不是视频生成完再配音。声音是生成的一部分。

```json
 {
        "ambient": [
          "暴雨",
          "脚踩积水声"
        ],
        "sfx": [
          {
            "time": 1.5,
            "sound": "急促脚步声"
          }
        ],
       "bgm": {
          "style": "低沉钢琴氛围",
          "intensity": "low"
        }
      },
```
