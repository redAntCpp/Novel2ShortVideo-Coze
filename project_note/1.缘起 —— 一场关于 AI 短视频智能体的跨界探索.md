# 缘起 —— 一场关于 AI 短视频智能体的跨界探索

说起来，已有好些年未曾提笔写过博客了。

步入职场后，分享的念头便渐渐淡了。一来是日常工作内容千篇一律，二来我所处的医疗信息行业，向来是技术保守的 “重灾区”，既无前沿技术的落地应用可探讨，也少了突破创新的动力，久而久之，便也习惯了按部就班的状态。本以为日子会这般平淡下去，却没想到今年初春的一件事，重新点燃了我沉寂已久的学习热情。

故事要从今年 3 月说起。我的表弟是某本科院校通信工程专业的学生，恰逢毕业设计阶段，不巧抽到了与 AI 相关的选题 —— 开发一个 AI 生成短视频的智能体。这可把他难住了，我原以为通信工程专业会侧重代码与网络方向，细问才知，核心课程更多围绕嵌入式展开，对于 AI 智能体这类领域，他几乎是零基础。实习结束后，他便找到我求助。坦白说，我对 AI Agent 领域也仅止于了解，从未实际上手过，这对我而言，也是一场不小的挑战。

不过也算恰逢其时，当时单位正推进 HIS 系统更换工作，作为该系统的负责人，因需求暂停和硬件环境筹备，我意外拥有了一段相对清闲的时间。借着这个契机，我决心主动接触一些前沿知识，跳出医疗信息这个提不起兴致的舒适区。更何况，我从小便有一个 “导演梦”，AI 视频技术的兴起，恰好打破了传统视频创作的技术与资金壁垒，让这个曾经遥不可及的梦想有了落地的可能。种种缘由叠加，我便下定决心，帮表弟把这个毕业设计做出来。

此次项目的核心目标：**用户上传一篇小说，智能体对小说进行分析和审核，通过后，对小说进行分集、剧本改编、分镜设计、角色设计、场景设计以及利用seedance 2.0 生成分镜视频，对分镜视频进行拼接、配音、添加字幕以及成片展示。**

在这里，先对这个项目做几点简单的说明：

1. 梳理需求时，我们接触到了多款自动化工作流平台，对比 n8n 和字节跳动的扣子平台后，最终决定以扣子作为核心工作流的开发载体；
2. 我的技术栈以 C/C++/C# 为主，此前从未接触过 Python，但该项目涉及大量 Python 代码开发，所有相关知识都是边学边用、现看现练；
3. 扣子平台同样是从零开始摸索，整个开发过程踩了不少坑，文中若有疏漏或不当之处，也欢迎各位读者指正；
4. 作为 AI 相关项目，开发过程中自然离不开各类 AI 工具的辅助，GPT、Gemini、豆包等都为我们提供了不少思路；
5. 项目中存在大量结构化内容，我们选择 JSON 作为节点间主要的数据传输协议，兼顾可读性与 Python 的处理便捷性；
6. 设计阶段参考了结构化分析方法，采用自顶向下的思路拆解需求、搭建框架；
7. 这一系列文章，更多是对项目开发过程的总结与归纳。受工作性质所限，无法保证持续更新，权当是一次个人经验的分享，各位看个乐便好。

接下来，我会在「扣子 & seedance 研讨室」这个专栏里，慢慢拆解这个 AI 短视频智能体项目的开发细节，从需求分析到落地踩坑，从技术选型到实际应用，把这段跨界探索的经历一一梳理出来。无论你是同样在摸索扣子平台的开发者，还是对 AI 短视频生成感兴趣的爱好者，希望这些分享能给你带来一点参考或启发。
